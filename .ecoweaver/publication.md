# Publication and Research Software Expectations

EcoWeaver software should be credible as research infrastructure, not just functional code.

## Research software principles

Prefer:

- reproducibility
- transparent assumptions
- citation-ready documentation
- testable workflows
- clear license information
- durable data formats
- explicit provenance
- small examples
- contributor-friendly setup

## JOSS-style expectations

For publication-oriented projects, ensure the repository has:

- clear statement of need
- installation instructions
- quick-start example
- tests and how to run them
- documentation for core functionality
- license
- citation metadata, such as `CITATION.cff`
- contributor guidance
- archived release plan

## Scientific claims

Avoid unsupported claims in documentation. Distinguish:

- implemented functionality
- planned functionality
- experimental functionality
- demonstrated results
- speculative future use

## Data and knowledge provenance

For evidence synthesis, ontology, or annotation tools, preserve:

- source artifact
- source fragment/span
- annotator or agent
- timestamp
- method/prompt/version where relevant
- uncertainty/review status
- license or governance constraints

## Agent instructions

When cleaning a repo for publication:

1. Inspect README, docs, tests, license, citation metadata, examples, and packaging.
2. Identify publication blockers.
3. Prefer actionable fixes over generic advice.
4. Do not exaggerate maturity.
