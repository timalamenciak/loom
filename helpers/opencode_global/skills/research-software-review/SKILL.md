# Research Software Review Skill

Use this skill when preparing a repository for collaborators, release, JOSS, Zenodo, or publication.

## Review areas

- statement of need
- installation
- quick start
- examples
- tests
- documentation
- license
- citation metadata
- contributor guidance
- release/archive readiness
- reproducibility
- provenance

## Output

Separate findings into:

1. Blocking issues
2. Important improvements
3. Nice-to-have improvements
4. Suggested concrete edits

Avoid generic advice. Prefer repository-specific recommendations.
