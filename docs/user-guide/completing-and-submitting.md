# Complete and submit an assignment

Complete individual edges first. Submit the assignment only after checking the
whole article annotation.

## Mark an edge complete

An edge begins as **Draft**. When its excerpts, nodes, and schema fields are
finished:

1. Save the edge.
2. Check its subject, object, relationship, and grounding excerpts.
3. Review every expanded and collapsed form section.
4. Select **Mark complete** in the edge form or **Mark done** in the graph
   panel.
5. Confirm that the edge badge changes to **Complete**.

Only a human annotator can make this transition. Loom does not silently promote
an automated proposal or unfinished draft.

## Final checklist

Before submitting, confirm that:

- all causal claims you intend to record have an edge;
- each edge points from the correct cause to the correct effect;
- every node and edge has appropriate grounding excerpts when text is
  available;
- **Source doc** has been checked and saved;
- required fields contain supported values;
- ontology identifiers came from actual search results;
- there are no accidental duplicate nodes or edges; and
- every finished edge displays **Complete**, not **Draft**.

## Submit for review

1. Select **Submit for review** in the session bar.
2. Read the confirmation message.
3. Confirm the submission.

Loom validates the entire graph against the schema version pinned to the
annotation. If validation succeeds, Loom:

- changes the assignment to **Submitted**;
- closes open work sessions;
- displays **Work submitted for review**; and
- returns you to **My queue**.

The submitted annotation becomes read-only.

!!! important
    Schema validation and edge completion are separate checks. Before
    submitting, inspect the graph panel yourself and make sure finished edges
    show **Complete**.

## If validation blocks submission

Loom keeps the assignment editable and reports that the graph did not validate
against its pinned schema. It also displays a limited set of validation
messages.

1. Read each displayed message.
2. Open the affected node, edge, or source document.
3. Correct and save the values.
4. Try **Submit for review** again.

Do not weaken or work around a required field merely to submit. If the article
cannot support a required value, ask the annotation lead how the project wants
that case handled.
