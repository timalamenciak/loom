# Troubleshooting

## My queue is empty

Ask a project administrator to confirm that:

- you are a member of the project; and
- the article is assigned to your Loom account.

## The article says Abstract or None

For **Abstract**, annotate only causal claims supported by the available
abstract. For **None**, ask an administrator to attach a PDF or provide
annotatable text.

## I cannot select text

Switch to the **Text** view. The **PDF** and **Markdown** views help with
reading and layout, but grounding excerpts are selected from canonical Text.

If the Text view is missing or its content is incomplete, note the article and
contact a project administrator. Do not create offsets by copying text from a
different file.

## An excerpt is missing from the form

1. Confirm that the excerpt appears in the excerpt bin.
2. Select its checkbox.
3. Choose **Use for node** or **Use for edge** again.
4. Confirm it is selected under **Grounding excerpts** before saving.

An excerpt already attached to another annotation may not be available for the
current item. Collect a separate source passage when each annotation needs its
own grounding.

## Ontology search has no suitable result

Try a shorter term, a synonym used by the article, or a broader concept. Read
result descriptions before choosing.

Do not select an incorrect term just to fill the field, and never type an
invented ontology identifier. Use free text or term suggestion only when the
form offers it. Otherwise, leave an optional field empty or ask the annotation
lead for project guidance.

## An annotation does not save

1. Keep the form open.
2. Read any message shown in or beside the form fields.
3. Correct invalid or missing required values.
4. Select the explicit save button again.
5. Confirm that the graph panel updates.

If a request-failed message appears, check your connection and try again. If
the problem continues, copy the non-sensitive error text and report the
article, project, action, and approximate time to an administrator. Do not send
passwords, session cookies, or other secrets.

## The timer says idle, hidden, or paused

- **idle**: interact with the page to resume active timing;
- **hidden**: return to the Loom browser tab; or
- **paused**: select **Resume**.

The timer intentionally excludes inactive tab-open time.

## I reversed the cause and effect

Select **Edit** beside the edge, correct the **Subject** and **Object**, then
select **Save edge**. If the edge already displays **Complete**, review it
carefully because editing does not automatically return it to Draft.

## I created a duplicate node

First edit any affected edges so that they reference the node you intend to
keep. Then delete the duplicate only after confirming that no needed edge will
be removed with it.

## Submission is blocked

Submission validates the complete graph against its pinned schema. Follow each
displayed validation message, save the corrections, and submit again. If the
article cannot support a required value, ask the annotation lead rather than
guessing.

## I cannot edit after submission

This is expected. Submitted and reviewed assignments are read-only. A reviewer
must return a submitted assignment before you can revise it.
