# Save and resume work

Saving a form, completing an edge, and submitting an assignment are different
actions. Saving does not automatically perform the next step.

## Save a new annotation

A new node or edge does not exist in the graph until you select its save
button.

- Select **Save node** when creating a node.
- Select **Save edge** when creating an edge.
- Select **Save source document** when recording article-level context.

After saving a node or edge, confirm that it appears in the graph panel. A new
edge is saved as **Draft**.

## Save changes to an existing annotation

When editing a previously saved node or edge, Loom auto-saves individual field
changes after a short delay. The save indicator changes to **Saved** with a
time after the server accepts the change.

Auto-save is a safeguard, not a replacement for the form's save button. Select
**Save node** or **Save edge** after meaningful edits so that Loom validates
the complete form and applies changes such as grounding excerpts and edge-node
selection together.

Before closing a form or leaving the page:

1. Select the form's save button.
2. Confirm that the graph panel updates.
3. Resolve any validation messages.

The close symbol at the top of a form closes it. It does not save a new node or
edge.

## Understand edge status after editing

Editing and saving a **Complete** edge does not currently change it back to
**Draft**. Recheck every section after editing because the edge continues to
display **Complete**.

## Leave and return safely

You can leave an **In progress** assignment and return from **My queue** later.
Saved nodes, edges, excerpts, and source-document context remain attached to
the annotation graph.

When you leave the workspace, Loom closes the current timing session. Opening
the workspace again starts or resumes an active session for the assignment.

## If a save fails

Do not assume the change was recorded. Keep the form open and follow the steps
in [Troubleshooting](troubleshooting.md#an-annotation-does-not-save).
