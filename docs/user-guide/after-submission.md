# After submission

A submitted assignment stays in **My queue**, but its annotation is read-only.
You can inspect the article, nodes, edges, and saved context without changing
them.

## Review statuses

Assignment and edge statuses describe different things:

- **Submitted** means the assignment has been sent to the reviewer.
- **Complete** means an annotator marked an individual edge finished.
- **Reviewed** means a reviewer has reviewed work at the relevant stage.
- **Gold** is the final accepted status for an adjudicated edge.

Annotators mark edges **Complete** and submit assignments. Reviewers or project
administrators control the later review and gold transitions.

## If an assignment is returned

A reviewer can return a **Submitted** assignment for more work. It then appears
in **My queue** with the status **Returned**.

To revise it:

1. Select **Open**.
2. Select **Start annotating**.
3. Review the requested changes provided through your project's review
   process.
4. Edit and save the relevant source context, nodes, or edges.
5. Recheck all affected edges, including edges that still display
   **Complete**.
6. Select **Submit for review** again.

Opening a returned assignment changes it to **In progress** and restores edit
access.

## Keep the audit trail intact

Revise the existing annotation when it represents the same claim. Do not make
replacement nodes or edges merely to hide a mistake. Loom records graph changes
for review and reproducibility.
