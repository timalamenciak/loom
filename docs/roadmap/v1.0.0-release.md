# v1.0.0 — Release Hardening

**Epic:** All · **Depends on:** v0.9.0

## Goal

Loom is production-ready: end-to-end tested against a non-CAMO schema, security-audited, documented for both admins and annotators, containerized for self-hosting, and accompanied by a migration guide for v0.x deployers.

## Implementation Steps

### Step 1 — End-to-end integration test

**File:** `tests/e2e/test_full_annotation_workflow.py`

A single test that walks the complete annotator journey for a **non-CAMO** schema — this is the schema-agnosticism smoke test.

The test uses Django `TestClient` only (no Playwright). It needs:
- A hand-written toy LinkML schema in `tests/fixtures/toy_schema.yaml` with 2 classes (ToyNode, ToyEdge) and 5 slots (name, description, strength/enum with 3 values, source_text, confidence/float)
- A small OBO file in `tests/fixtures/toy_ontology.obo` with 3 terms
- A test PDF or minimal document bytes (reuse existing test fixtures if present)

### Step 2 — Security review

Check the following and fix any findings:

**File upload validation**
- Only allow `.yaml`, `.yml`, `.obo`, `.owl`, `.pdf` at upload endpoints
- Validate file size (reject > 50MB)
- Magic-byte check on uploaded files (detect mislabeled content): YAML must parse, OBO must have `format-version:` header, PDF must start with `%PDF`

**Authentication + authorization**
- Every `/admin/*` view must be `login_required` + `is_staff`
- Every annotation view must be `login_required` + project member
- The LLM review queue must check the annotator is assigned to the project

**XSS**
- Ontology term labels and slot descriptions are displayed in templates — confirm they use `{{ value }}` (auto-escaped) not `{{ value|safe }}`
- JSON embedded in `<script>` tags must use `json_script` template filter, not `|safe`

**API keys**
- Confirm `ProposerConfig.api_key_env_var` stores only the env var name
- Search for `api_key` in the DB schema and confirm no plaintext key columns exist

**CSRF**
- All non-GET views use Django's CSRF middleware (on by default, but confirm no `@csrf_exempt` decorators exist outside of explicitly justified cases)

### Step 3 — Performance sanity check

The following operations must complete in under 2 seconds for a project with 500 documents and 2000 edges:

- Annotation page load (form_spec generation is LRU-cached — verify cache is hit on second load)
- Ontology search (pg_trgm index exists on `synonym_labels` — verify with `EXPLAIN ANALYZE`)
- Proposal review queue load (check for N+1 queries — `select_related` must cover subject/object/graph/document)
- Export (serializer must not issue per-edge queries — verify with Django Debug Toolbar or `assertNumQueries`)


### Step 4 — Documentation

Update or create the following docs (in `docs/`):

**`docs/admin-guide.md`** — tasks admin users perform:
- Uploading and activating a schema
- Configuring the form builder (layers, hidden slots, widget overrides)
- Loading and browsing ontologies
- Mapping ontologies to slots
- Configuring the LLM pipeline (ProposerConfig, API key, few-shot selection)
- Reviewing update notifications and applying updates

**`docs/annotator-guide.md`** — tasks annotators perform:
- Creating nodes and edges from highlighted text
- Using ontology autocomplete
- Reviewing and accepting LLM proposals
- Understanding work session tracking

**`docs/migration-v0-to-v1.md`** — steps for upgrading from v0.x:
1. `python manage.py migrate` (runs all new migrations)
2. `python manage.py load_schema config/schema/camo-0.4.0.yaml --activate` (re-activate if needed)
3. `python manage.py load_ontology --all` (re-loads with new schema)
4. `python manage.py check_all_updates` (optional: check for upstream updates)
5. Verify existing `CausalGraph` rows still export correctly: `python manage.py validate_graph --all`

### Step 5 — Production Docker compose

**File:** `docker-compose.prod.yml`

Services: `web` (Django + Gunicorn), `db` (Postgres 15 with `pg_trgm` extension), `cache` (Redis for Django cache, replacing thread-based update-check cache).

Update `loom/settings/` to use `django-redis` for cache backend in production. Add a `healthz` endpoint (simple `200 OK` view) for container health checks.

Create `.env.example` documenting all required environment variables: `SECRET_KEY`, `DATABASE_URL`, `REDIS_URL`, `ALLOWED_HOSTS`, `LLM_API_KEY` (if LLM enabled).

### Step 6 — Migration readiness script

**File:** `scripts/check_migration_readiness.py`

A script run in CI (and by admins before upgrading) that checks:
- Active SchemaVersion exists
- At least one OntologySnapshot exists
- No `Node` or `Edge` row has `graph__schema_version=None`
- All existing graphs export without validation errors (dry-run the serializer)
- Exits 0 if all checks pass, 1 with descriptive error messages if not
