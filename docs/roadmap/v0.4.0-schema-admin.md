# v0.4.0 — Schema Admin Foundation

**Epic:** E1: Schema · **Depends on:** v0.3.0 · **Unlocks:** v0.5.0 (Form Builder), v0.8.0 (LLM Pipeline)

## Goal

Admins can upload, activate, and browse LinkML schemas from the browser. The `loom_ui.yaml` UI config moves to the database — stored per `SchemaVersion` and optionally overridden per `Project` — so it can be edited per-deployment without file access. Projects can independently select different schema versions.

## Baseline

- `apps/schemas/schema_engine.py` — `LoomSchemaView.form_spec()` is fully implemented; it reads `loom_ui.yaml` via `ui_config.py`.
- `apps/schemas/models.py` — `SchemaVersion` model exists (YAML, SHA-256, `is_active` flag).
- `apps/schemas/management/commands/load_schema.py` — works but requires CLI.
- `config/loom_ui.yaml` — 209-line production config; must remain the fallback default.

No admin views for schemas exist yet.

## Implementation Steps

### Step 1 — `SchemaUIConfig` model

**File:** `apps/schemas/models.py`

Add a model that mirrors `loom_ui.yaml` structure but is DB-backed:

- `schema_version` — FK to `SchemaVersion` (on_delete=CASCADE)
- `project` — nullable FK to `Project` (on_delete=CASCADE); null = applies to all projects using this schema version
- `layers` — JSONField (list of `{name, slots, collapsed_default}`)
- `ontology_routing` — JSONField (dict)
- `widget_overrides` — JSONField (dict, slot → widget type)
- `globally_hidden_slots` — JSONField (list of slot names)
- `slot_help_text` — JSONField (dict, slot → placeholder string)

Add a classmethod `SchemaUIConfig.for_schema_version(sv, project=None)` that:
1. Checks for a project-level record (`schema_version=sv, project=project`) first
2. Falls back to a schema-level record (`schema_version=sv, project=None`)
3. Falls back to parsing `config/loom_ui.yaml` into a transient (unsaved) `SchemaUIConfig` instance

This ensures all existing behavior is preserved with zero migration risk.

**Add migration.** Add a `Meta.constraints` unique_together on `(schema_version, project)`.

### Step 3 — Schema upload view

**File:** `apps/schemas/views.py` (create if missing), `loom/urls.py`

`SchemaUploadView` at `GET/POST /admin/schemas/upload/`:
- GET: render `templates/schemas/upload.html` with a file input and a "Version string" read-only field (populated client-side via HTMX peek if desired, or just shown after POST)
- POST: read uploaded bytes, validate with `linkml_runtime.SchemaView`, extract `schema.version`, compute SHA-256, call existing load_schema logic to create `SchemaVersion`
- On success: redirect to schema detail
- On failure: re-render form with the linkml parse error
