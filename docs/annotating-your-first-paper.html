Overview

What you are annotating

Loom captures causal claims from scientific papers. Each claim you find in a paper becomes a causal edge connecting two nodes. A node represents one ecological entity (a species, a process, an abiotic variable) described with a measurable attribute and a direction of change.

The annotation schema is called CAMO — Causal Mosaic. It asks you to decompose every causal sentence into two ELMO nodes and then characterise the causal relationship across four layers of increasing depth.

The goal in one sentence: find a sentence in the paper that asserts X causes Y, decompose X and Y into nodes, then fill in the edge form to record how strong the claim is, what philosophical account it rests on, and what evidence supports it.
Step 1

Finding your papers

After logging in you land on the Loom home page. Click My queue in the top navigation to see every paper currently assigned to you.

Each row shows the paper title, the project it belongs to, whether full-text (PDF) or abstract-only text is available, and your current status on that assignment:

  • Assigned — not yet opened
  • In progress — you have started but not submitted
  • Submitted — sent for review

Click Open to go to the document detail page, then click Annotate to enter the annotation workspace.

Abstract-only documents: if the row shows Abstract, you will only see the abstract text in the workspace — not the full paper. Annotate what you can from the abstract alone.
Step 2

Understanding the annotation interface

The annotation workspace is a two-pane layout that fills the browser window.

🖼
Screenshot
Full annotation workspace — session bar across the top, paper text on the left, graph panel + form panel on the right

Session bar (top strip)

The bar at the top shows the paper title, your active annotation time, and the Submit button. The timer counts only the seconds when the tab is in the foreground and you have been recently active — it automatically pauses when you switch away. You can also pause it manually with Pause.

Left pane — paper text

For PDF-backed papers, the workspace opens on selectable text extracted from the PDF. Use the Text from PDF and PDF buttons to switch views. Select a passage and click Add to excerpt bin. You can collect several passages when a claim is spread across sentences.

Right pane — graph panel and form panel

The right side has two sections stacked vertically:

  • Excerpt bin (top) — keeps selected passages pinned while you work. Check one or more excerpts and choose Use for node or Use for edge.
  • Graph panel (middle) — lists every node and edge you have created for this paper. Use the + Node and + Edge buttons to create items without grounding excerpts.
  • Form panel (bottom half) — the active node or edge form appears here. Submitting a form refreshes the graph panel list automatically.
Step 3

Creating nodes: the ELMO decomposition

A node in CAMO represents one side of a causal claim. Before you can create an edge you need at least two nodes — one for the cause and one for the effect.

Nodes follow the ELMO decomposition: every node has an Entity, an attribute it eLates to (the measurable variable), and a directiOn of change.

Example: the sentence "Buckthorn increases canopy cover, suppressing native seedlings" gives two nodes:
Node 1 — buckthorn / canopy cover / increases
Node 2 — native seedlings / abundance / decreases

How to create a node

  1. Highlight one or more passages that mention the entity. Add each passage to the excerpt bin.
  2. Check the excerpts that ground this node and click Use for node, or click + Node to create an ungrounded node.
  3. Fill in the node form that appears in the form panel:
🖼
Screenshot
Add Node form — name field, Entity type dropdown, Entity term search, Variable attribute, Direction dropdown
FieldWhat to enter
Name A short, human-readable label for this node. Will appear in the graph panel list and in edge summaries. Keep it concise: "buckthorn cover ↑" is more scannable than the full ELMO sentence.
Entity type Broad category: biotic (organism), abiotic (temperature, light), process (predation), anthropogenic (land use), compound (nutrient).
Entity term Start typing to search the ontology (NCBITaxon for organisms, ENVO for environments). Select the best match; leave blank if nothing fits.
Variable attribute The measurable property of the entity that changes. Search PATO (phenotype and trait ontology) for standardised terms such as "abundance", "biomass", "diversity".
Direction increases, decreases, or neutral (presence/absence). Use not addressed if the text does not specify.
Notes (optional) Any clarifying remarks about this node.
  1. Click Save node. The node appears in the graph panel's Nodes list immediately.
  2. Repeat for the second node (the other side of the causal relationship).
Reuse nodes across edges. If the same entity appears as both subject and object in different causal claims, create one node and reference it in multiple edges. Do not duplicate it.
Step 4

Creating an edge

An edge is the causal link between two nodes. Once you have at least two nodes, you can create an edge.

  1. Add every passage supporting the causal claim to the excerpt bin. Check the relevant excerpts and click Use for edge — or click + Edge to create an ungrounded edge.
  2. The edge form appears in the form panel. It is divided into four collapsible layers (described below). Fill them in from top to bottom.
🖼
Screenshot
Add Edge form — Layer 1 open (subject/object dropdowns, predicate, claim strength, original sentence), Layers 2–4 collapsed below
  1. Select the Subject node (the cause) and the Object node (the effect) from the dropdowns. Both dropdowns list every node you have created for this paper.
  2. Complete Layers 1–4 (see next section).
  3. Click Save edge. The edge appears in the graph panel's Edges list with a Draft badge.
Step 5

The four layers of a causal edge

CAMO records four complementary perspectives on every causal claim. The first two layers are always visible; Layer 3 is collapsed by default (it contains the detailed causal features). Layer 4 records the evidence base.

Layer 1 — Claim & Predicate
Subject · Object · Predicate · Claim strength · Original sentence
Layer 2 — Philosophical Account
Philosophical account · Certainty grade
Layer 3 — Causal Features (collapsed — expand to fill)
Mediation · Moderation · Context · Effect magnitude · Temporal extent · Study site · Mechanism known · Reversibility · Ecological level
Layer 4 — Evidential Basis
Study design · Sample size qualifier · Effect size · Statistical significance

Layer 1 — Claim & Predicate

This is the heart of the annotation. It captures what the paper claims happens and how strongly it is claimed.

FieldGuidance
Predicate The directed causal verb linking subject to object.
Claim strength How strongly is the claim stated? See linguistic cues below.
Original sentence Paste (or it auto-fills from your highlight) the exact sentence that expresses the causal claim. This is your evidence anchor.

Predicates at a glance:

PredicateMeaningExample
positively_regulatesSubject increases the object's attribute"Nitrogen addition increases plant biomass"
negatively_regulatesSubject decreases the object's attribute"Predation reduces prey density"
regulatesSubject affects object; sign unspecified"Temperature affects metabolic rate"
contributes_toSubject is one of several factors"Herbivory contributes to seedling mortality"
is_facilitated_byObject is enabled by subject"Germination is facilitated by soil moisture"
is_inhibited_byObject is suppressed by subject"Photosynthesis is inhibited by shade"

Claim strength — linguistic cues:

StrengthCue words in the text
necessary_conditionrequires, must, cannot without, only if, prerequisite
sufficient_conditionsufficient to, causes by itself, enough to
strong_tendencyreliably, consistently, strongly, robustly tends to
tendencytends to, generally, usually, often, commonly
weak_tendencymay, can, sometimes, under certain conditions
not_addressedNo explicit signal — use when genuinely ambiguous

Layer 2 — Philosophical Account

What makes the paper's causal claim causal? This layer asks you to identify the philosophical framework the authors (implicitly or explicitly) rely on.

AccountWhen to use it
regularityThe claim is a law-like pattern: "whenever A, then B"
counterfactualThe claim rests on "but for X, Y would not have occurred"
mechanisticA specific pathway or mechanism is named (via, through, mediates)
interventionistBased on a manipulation: "removing grazers leads to…"
probabilisticFramed as a change in probability or risk
not_addressedNo clear signal

Certainty grade is a 0–1 slider expressing your confidence that this philosophical account applies. 0.9 means "clearly fits"; 0.5 means "plausible but uncertain".

Layer 3 — Causal Features

Layer 3 contains fifteen detailed fields that characterise the causal relationship. It is collapsed by default — click the summary bar to expand it. Fill in as much as the paper supports; leave fields as not_addressed when the paper provides no information.

Key fields to prioritise:

  • Mediation — is there a named mediating variable? (e.g. "via light competition")
  • Moderation — does a third variable change the direction or strength of the effect?
  • Ecological level — individual, population, community, ecosystem, landscape, or global?
  • Mechanism known — yes / no / not addressed
  • Reversibility — can the effect be reversed if the cause is removed?

Layer 4 — Evidential Basis

What type of study produced this evidence?

Study designMeaning
experimentControlled experiment with random assignment
quasi_experimentNatural or quasi-experiment (BACI, before-after, gradient)
observationalCorrelational; no manipulation
meta_analysisQuantitative synthesis across primary studies
reviewNarrative or systematic review
modelMathematical, statistical, or simulation model
Step 6

Completing edges and submitting your work

Marking an edge complete

An edge starts as Draft. Once you are satisfied with all four layers, mark it done by clicking ✓ complete in the edge row in the graph panel (or the ✓ Mark complete button at the bottom of the edge form). The badge changes to Complete.

Draft → Complete → Reviewed → Gold

Reviewed and Gold are set by a project reviewer (adjudicator) — you do not need to worry about those.

Mark every edge complete before submitting. Draft edges signal to the reviewer that something is unfinished. If you truly cannot complete a field, leave it as not_addressed and mark the edge complete anyway.

Submitting your annotations

When you have processed all the causal claims in the paper:

  1. Check the graph panel — every edge should show Complete.
  2. Click Submit in the session bar. A confirmation dialog will appear.
  3. Confirm. Your assignment status changes to Submitted and the paper moves to the reviewer's queue.
You can still view your annotations after submitting, but you cannot edit them unless the reviewer returns the assignment to you.

Tips

Tips & common questions

How many edges should I create per paper?

Annotate every distinct causal claim you can find. A short methods paper might yield 2–3 edges; a review article can yield 15 or more. Aim for precision over quantity — one well-filled edge beats two half-filled ones.

What counts as a causal claim?

Look for verbs like increases, decreases, drives, causes, suppresses, facilitates, inhibits, promotes, leads to, results in, reduces. Correlational statements ("X was associated with Y") are weaker — annotate them with weak_tendency and observational study design rather than skipping them.

Do I annotate the methods or the results?

Annotate causal claims wherever they appear: introduction, results, discussion, abstract. The original sentence field preserves exactly where the claim came from.

The ontology search returns no results

Try a shorter or more general search term. The local index is preloaded from a snapshot — if a term genuinely does not exist, leave the field blank and note the preferred term in the node's Notes field. Do not invent terms.

I made a mistake in a node

Click edit next to the node in the graph panel. Update the fields and click Save node. Changes are applied immediately to all edges that reference this node.

I made a mistake in an edge

Click edit next to the edge in the graph panel. If the edge is already Complete, editing it reverts it to Draft — remember to mark it complete again when you are done.

Can I annotate the same claim from two angles?

Yes. If a sentence supports both a primary causal claim and a secondary mediating pathway, create separate edges — one for each distinct claim. Annotate the mediating pathway in the Mediation fields of the primary edge, and also as its own edge if it is stated explicitly enough.

Session timer paused itself

The timer pauses automatically after 90 seconds of inactivity. Move the mouse, click anywhere on the page, or press a key to resume it. You can also toggle it manually with the Pause button.
